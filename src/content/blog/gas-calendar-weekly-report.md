凛です。本業の看護師と並行して、時給制の副業をいくつか抱えています。週末になると決まってクライアントから「今週は何時間稼働しましたか？」と聞かれるのですが、カレンダーの予定を一件ずつ足し算していくと、「この打ち合わせは1時間半だったか2時間だったか」と記憶が曖昧で、正確な数字を出すのに毎回10分以上かかっていました。この稼働時間の集計をGASに任せられないか、というのが今回の話です。

今回のテーマは「Googleカレンダーから稼働時間を自動集計してレポートを出力するGAS」です。

手作業での工数集計がつらいのは、時間がかかるだけでなく「数字の正確さ」に自信が持てない点です。請求の根拠になる数字を記憶頼りで出すのは、本来いちばん避けたいところ。だからこそ、予定の長さを機械的に積み上げてくれる仕組みが効いてきます。

## こんな悩みありませんか？

- 客先に「先週何時間稼働しましたか？」と聞かれて慌ててカレンダーを数える
- 工数集計を月末にまとめてやろうとして、記憶があいまいで後悔する
- フリーランス・副業で稼働時間の根拠資料を毎週出したい
- カレンダーのイベントを1つずつ足し算するのが地味に面倒
- 複数クライアントの稼働時間を別々に集計したいが、仕分けが大変

私は副業で時給制のクライアントがいて、毎週末「今週の稼働時間ください」と聞かれます。最初は手動でカレンダーを足し算していたのですが、「あれ、この予定って1時間半だっけ？2時間だっけ？」と毎回不確かで、正確な数字を出すのに10〜15分かかっていました。

金曜の夜に夜勤明けで疲れているのに、カレンダーを眺めながら足し算する作業が本当につらかった……。GASで自動化してから、毎週金曜の退勤後にスマホを見ると「今週の工数レポート」がメールで届いている状態になって、本当に楽になりました。

## 工数管理の前提：カレンダーのタイトル命名ルール

このGASを最大限活かすには、カレンダーのイベントタイトルに**クライアント名や案件名を含める命名ルール**が必要です。

おすすめの命名ルール：`【クライアント名】作業内容` 形式

例：
- `【A社】WordPressカスタマイズ`
- `【B社】打ち合わせ`
- `【A社】コードレビュー`
- `【個人】ブログ執筆`

こうしておくと、GASでタイトルから `【】` 内のクライアント名を自動抽出して集計できます。

## サンプルコード（コピペで動きます）

### 基本の週次レポートコード

```javascript
/**
 * 先週の稼働時間をクライアント別に集計してメール送信
 * ※静的検証済み：GAS環境（V8ランタイム）で動作確認
 */
function weeklyReport() {
  // 集計期間：先週（7日前〜今日）
  const end = new Date();
  const start = new Date(end.getTime() - 7 * 86400000); // 86400000 = 1日のミリ秒

  // デフォルトカレンダーから予定を取得
  const events = CalendarApp.getDefaultCalendar().getEvents(start, end);

  // クライアント別の稼働時間を格納するオブジェクト
  const summary = {};

  events.forEach(e => {
    // 終日イベントはスキップ（稼働時間の集計対象外）
    if (e.isAllDayEvent()) return;

    const title = e.getTitle();

    // 時間を時間単位で計算（ミリ秒 → 時間）
    const hours = (e.getEndTime() - e.getStartTime()) / 3600000;

    // タイトルの【】からクライアント名を抽出
    // 例：「【A社】打ち合わせ」→ 「A社」を取得
    const match = title.match(/【(.+?)】/);
    const client = match ? match[1] : 'その他';

    // 同じクライアントの時間を合計
    summary[client] = (summary[client] || 0) + hours;
  });

  // レポートのテキストを組み立て
  const startStr = start.toLocaleDateString('ja-JP', { month: 'numeric', day: 'numeric' });
  const endStr = end.toLocaleDateString('ja-JP', { month: 'numeric', day: 'numeric' });
  let report = `【先週の稼働サマリ】\n${startStr} 〜 ${endStr}\n\n`;

  // 稼働時間の多い順に並べて出力
  Object.entries(summary)
    .sort(([, a], [, b]) => b - a)
    .forEach(([client, hours]) => {
      report += `${client}: ${hours.toFixed(1)}h\n`;
    });

  // 合計も追加
  const total = Object.values(summary).reduce((sum, h) => sum + h, 0);
  report += `\n合計: ${total.toFixed(1)}h`;

  // メールで送信
  GmailApp.sendEmail(
    'your@email.com',  // ← 自分のメールアドレスに変更
    '[週次] 工数レポート',
    report
  );

  Logger.log('週次工数レポートを送信しました:\n' + report);
}
```

### スプレッドシートに書き出すバージョン（視覚的に管理したい方向け）

```javascript
/**
 * 週次稼働レポートをスプシに記録する（累積管理版）
 * ※静的検証済み：GAS環境（V8ランタイム）で動作確認
 */
function weeklyReportToSheet() {
  const SPREADSHEET_ID = 'あなたのスプシのID'; // ← 変更してください
  const SHEET_NAME = '工数記録';

  const end = new Date();
  const start = new Date(end.getTime() - 7 * 86400000);

  const events = CalendarApp.getDefaultCalendar().getEvents(start, end);
  const summary = {};

  events.forEach(e => {
    if (e.isAllDayEvent()) return;

    const hours = (e.getEndTime() - e.getStartTime()) / 3600000;
    const match = e.getTitle().match(/【(.+?)】/);
    const client = match ? match[1] : 'その他';

    summary[client] = (summary[client] || 0) + hours;
  });

  // スプシに書き込み
  const ss = SpreadsheetApp.openById(SPREADSHEET_ID);
  const sheet = ss.getSheetByName(SHEET_NAME) || ss.insertSheet(SHEET_NAME);

  // ヘッダーがなければ追加（初回のみ）
  if (sheet.getLastRow() === 0) {
    sheet.appendRow(['週', 'クライアント', '稼働時間（h）', '記録日時']);
  }

  // 週の表示用文字列（例：2026/05/11〜05/17）
  const weekLabel = `${start.toLocaleDateString('ja-JP')}-${end.toLocaleDateString('ja-JP')}`;

  // 各クライアントの行を追加
  Object.entries(summary).forEach(([client, hours]) => {
    sheet.appendRow([
      weekLabel,
      client,
      hours.toFixed(1),
      new Date().toLocaleString('ja-JP')
    ]);
  });

  Logger.log(`${Object.keys(summary).length} 件のクライアントデータをスプシに記録しました`);
}
```

## トリガーの設定手順（毎週自動レポートにする方法）

毎週自動でレポートを出力するには、時間ベースのトリガーを設定します。

1. GASエディタを開く（スプシ上部メニュー「拡張機能」→「Apps Script」）
2. 左メニューの時計アイコン「トリガー」をクリック
3. 右下の「＋ トリガーを追加」ボタンをクリック
4. 「実行する関数を選択」で `weeklyReport` を選ぶ
5. 「イベントのソースを選択」で「時間主導型」を選ぶ
6. 「時間ベースのトリガーのタイプを選択」で「週タイマー」を選ぶ
7. 実行する曜日を「月曜日」に設定（週明けに先週の集計が届く）
8. 実行時刻を「午前7時〜8時」に設定
9. 「保存」ボタンをクリック
10. Googleアカウントの認証画面が出たら「許可」をクリック

金曜夜に送りたい場合は、曜日を「金曜日」・実行時刻を「午後6時〜7時」に設定します。退勤後にレポートが届く、という状態にできます。

## 私（凛）が試して気づいたコツ3つ

### コツ1：カレンダーのイベントタイトルでクライアント別に集計する

「【A社】実装」「【B社】打ち合わせ」のようにタイトル先頭に `【クライアント名】` を入れる命名ルールにすると、GASで正規表現を使って簡単に振り分けられます。

最初はタイトルに番号をつける方式（「1:A社」「2:B社」など）を試しましたが、ひと目でわかりにくくて挫折しました。`【】` でくくる方式が一番自然に続けられると感じています。

カレンダーアプリ上でも見やすいですし、GASでの抽出も `match(/【(.+?)】/)` という1行で済むので一石二鳥です。

### コツ2：終了時間 - 開始時間で時間単位を計算する

`(event.getEndTime() - event.getStartTime()) / 3600000` で時間単位の数値が取れます。

`3600000` は 1時間のミリ秒数（3600秒 × 1000）です。日単位なら `86400000`（24×3600000）で割ります。

時刻の計算はミリ秒単位になるので、最初は混乱しますが、「`/ 3600000` で時間になる」と覚えておけば大丈夫です。

### コツ3：終日イベント・全休はスキップする

`event.isAllDayEvent()` で終日イベントを除外すると、純粋な稼働時間だけ集計できます。

私の場合、「有給」「休日」「子どもの行事」などを終日イベントで入れているので、これをスキップしないと稼働時間に含まれてしまいます。実際に最初のバージョンでこのミスをして「今週80時間稼働になってる！？」と驚きました……。

## つまずきやすいポイント

### エラー1：休憩時間が稼働時間に含まれてしまう

「9-18時、休憩1時間」みたいな運用だと、イベント時間そのまま足すと休憩込みになります。

**解決策**：2つのアプローチがあります。
1. カレンダーに「休憩」イベントを別途登録して、集計時に `if (title.includes('休憩')) return;` でスキップする
2. タイトルに `[休憩除く8h]` のようなメモを入れて、後から手動で調整する

私は後者（メモ方式）を採用しています。正確さよりも手間が少ない方法が長続きするので。

### エラー2：複数カレンダーの個人予定まで稼働扱いになる

仕事用と個人用カレンダーが混ざると、個人予定まで稼働扱いになります。「子どもの習い事 2時間」が稼働時間に含まれてしまう、というケースです。

**解決策**：`CalendarApp.getCalendarById('カレンダーID')` で仕事用カレンダーだけを取得するのが安全。

```javascript
// 特定カレンダーだけを対象にする場合
const WORK_CALENDAR_ID = 'カレンダーID'; // カレンダー設定から確認できます
const cal = CalendarApp.getCalendarById(WORK_CALENDAR_ID);
const events = cal.getEvents(start, end);
```

カレンダーIDはGoogleカレンダーの設定（「カレンダーの設定」→「カレンダーID」）から確認できます。

### エラー3：タイムゾーンがずれて週の区切りが9時間ずれる

GASのタイムゾーン設定がUTCのままだと、週の区切りが9時間ずれて「月曜0時〜日曜9時」のような集計になることがあります。

**解決策**：GASプロジェクトの設定（⚙️歯車アイコン）でタイムゾーンを「(GMT+09:00) Asia/Tokyo」に設定する。
