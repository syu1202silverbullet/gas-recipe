「毎朝9時にスプレッドシートを集計したい」「フォーム送信されたら即通知を飛ばしたい」。こうした**定期実行・イベント駆動実行**を可能にするのがGASの**トリガー**機能です。

この記事では、トリガーの全種類と設定手順を、実用例つきで完全解説します。

## トリガーとは？

簡単に言うと「**書いた関数を、指定タイミングで自動実行する仕組み**」です。

- 毎日朝8時に自動実行
- フォーム送信時に即実行
- スプレッドシートが編集されたら即実行
- 10分おきに実行

これらすべてが**無料**で使えます。

## トリガーの種類

GASのトリガーは大きく2タイプあります。

### 時限トリガー（時間ベース）

一定の時刻・間隔で実行。

| 間隔 | 用途 |
|---|---|
| 毎分〜30分 | リアルタイム監視（株価・在庫チェック） |
| 毎時 | 定期集計 |
| 毎日（時刻指定） | 日次レポート |
| 毎週（曜日＋時刻） | 週次まとめ |
| 毎月（日＋時刻） | 月次処理・バックアップ |

### イベントトリガー

特定のアクションで実行。

| イベント | 用途 |
|---|---|
| onOpen | スプシを開いた時（カスタムメニュー設定など） |
| onEdit | セル編集時（即通知など） |
| onFormSubmit | フォーム送信時（自動返信・Slack通知） |

## 時限トリガーの設定手順

例：毎朝8時に `dailyReport` という関数を自動実行する設定。

### ステップ1: GASエディタを開く

対象のスプレッドシートから「**拡張機能 → Apps Script**」。

### ステップ2: 関数を書く

```javascript
function dailyReport() {
  Logger.log('毎朝8時に実行されるはずの処理');
  // ここに実際の処理を書く
}
```

### ステップ3: トリガーアイコンをクリック

左サイドバーの「⏰ トリガー」（時計アイコン）をクリック。

### ステップ4: トリガーを追加

右下の「**+ トリガーを追加**」をクリックし、以下を設定：

- 実行する関数: `dailyReport`
- 実行するデプロイ: `Head`
- イベントのソース: `時間主導型`
- 時間ベースのトリガーのタイプ: `日付ベースのタイマー`
- 時刻: `午前8時〜9時`

保存すれば完了。翌朝8〜9時の間に自動で `dailyReport` が動きます。

## イベントトリガーの設定手順

例：スプレッドシートのセルが編集されたら、Slackに通知。

```javascript
function onEdit(e) {
  const sheet = e.source.getActiveSheet();
  const range = e.range;
  // ここでSlack通知など
  Logger.log('編集通知: ' + sheet.getName() + ' の ' + range.getA1Notation());
}
```

`onEdit` という名前の関数は、**名前だけで自動的にトリガーが紐付く**簡易実装。複雑な設定にする場合はトリガー管理画面で明示的に登録します。

## 抑えておくべき3つのポイント

### ポイント1: タイムゾーンを「Asia/Tokyo」に

デフォルトが米国になっていると「9時に設定したのに深夜0時に動く」事故が起きます。

**「プロジェクトの設定」→「タイムゾーン」→ Asia/Tokyo** に変更を忘れずに。

### ポイント2: エラー通知を「即時」にする

トリガーは裏で勝手に動くので、失敗しても気付きません。

トリガー追加画面の **「エラー通知設定」を「今すぐ通知を受け取る」** にしておくと、失敗時にGmailが飛んできます。

### ポイント3: 同じトリガーを二重登録しない

ボタンを連打するとトリガーが複数作られ、同じ処理が2回・3回走ります。

**トリガー一覧画面で、同名関数のトリガーが1つだけになっているか**を定期的にチェックしましょう。

## よくあるエラーと対処

### 6分制限に引っかかる（"Exceeded maximum execution time"）

1回の実行が6分以内でないといけないルール。

対処法：
- 処理をバッチ分割（100件ずつ→次回に続きから）
- `PropertiesService` で進捗を保存

### 権限エラー

新しいAPIを使い始めた時は、手動で1度実行して権限許可を済ませてからトリガー化する必要があります。

## 応用：副業・個人事業で使える組み合わせ

- **毎朝7時**: 前日の売上をスプシ集計 → LINEで通知
- **毎週月曜8時**: 今週のタスクをTODOリストから抽出 → Slack投稿
- **毎月1日0時**: 前月データを別シートへ退避 → 新月テンプレ生成
- **5分おき**: 在庫数をチェック → 閾値割れでメール通知

看護師の私の使い方は「毎朝6時に、子供の学校行事・自分の勤務予定・家族の予定を集約してLINEへ送信」でした。朝慌ただしい時間に情報を1通にまとめてくれるのは、本当に助かっています。

## まとめ

トリガーは「一度書けば永遠に動く」自動化の核です。設定自体は1分で終わるので、**作った関数には必ずトリガーを紐付ける**習慣をつけると生産性が桁違いになります。
