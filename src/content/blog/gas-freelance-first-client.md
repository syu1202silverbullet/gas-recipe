## こんな悩みありませんか？

- 「勉強は終わったのに、応募ボタンが怖くて押せない」
- 「実績ゼロでも取れる案件って本当にあるの？」
- 「提案文・ポートフォリオ・プロフィール、全部何を書けばいいか分からない」

こんにちは、凛です。現役ナース／の母／副業GAS歴3年です。

私も最初の応募は1ヶ月悩みました。でも、**最初の1案件が取れると、次の案件は一気に楽になります**。この記事では、私と副業仲間たちが実際に初案件を取った提案テンプレ・ポートフォリオ構成・プロフィール文例を、できるだけ生々しくまとめます。

※2026年4月時点の個人の感想・実体験ベースです。案件獲得・収益を保証するものではありません。

## 最初の1案件までの全体像

| フェーズ | やること | 目安期間 |
|---|---|---|
| ① 土台 | プロフィール・ポートフォリオ整備 | 3〜5日 |
| ② 応募 | 1日1〜3件の提案を3週間 | 2〜3週間 |
| ③ 面談 | チャット・ビデオで要件確認 | 1〜3日 |
| ④ 納品 | 実装・納品・検収 | 案件次第 |
| ⑤ 評価獲得 | レビュー依頼・リピート交渉 | 1週間 |

合計で**4〜6週間**が、0→1のリアルな期間感です。

## STEP1：ポートフォリオを「3本」用意する

### なぜ3本か

1本だと「偶然」と見られ、3本あると**カバー範囲が伝わる**からです。私が最初に作った3本は以下でした。

| # | 題材 | 使う技術 |
|---|---|---|
| 1 | スプシに日報を書くと自動で週次集計 | スプシ関数＋GAS集計 |
| 2 | Gmailで特定ラベル付きメールの添付をDriveに自動保存 | Gmail API＋Drive API |
| 3 | Googleフォーム回答をスプシ＋Slackに二重通知 | FormトリガーUrlFetchApp |

### 掲載形式

GitHubにコードを置き、READMEに以下をセットで書きます。

- 何の課題を解決したか
- スクショ（Before/After）
- 所要時間
- 想定される応用先

これが**そのまま提案文の素材**になります。

### GitHubがなくてもOK

NotionやGoogleドキュメントで公開してもOKです。大事なのは**URL1本でまとめて見せられる状態**にすること。

## STEP2：プロフィール文のテンプレ

クラウドワークス／ランサーズのプロフィール欄は、**頭3行で読まれるかが決まります**。私のプロフィールの構造を公開します。

### 見出し（1行目）

「看護師×GASで業務自動化／スプシ・Gmail・カレンダー連携が得意です」

→ **属性＋得意領域**を1行で明示。

### 2〜3段落目：経歴

「現役看護師として10年、複数の病院でシフト管理・記録業務に携わってきました。現場で『この集計、手作業じゃなくていいのに』と感じる場面が多く、独学でGASを学習。自分の職場の業務を月20時間削減した経験を、他の現場の効率化にも活かしたいと思い、副業を始めました。」

→ **痛みへの共感**がある属性の人は強い。

### 得意な業務

- Googleスプレッドシートの集計・自動化
- Gmail自動返信・振り分け・ラベル管理
- Googleカレンダー予定一括登録
- Googleフォーム回答の自動加工・通知

### 稼働時間

「平日夜20時〜23時、土日の午前中を中心に対応可能です。夜勤明けの翌日のみ返信が遅れる場合があります。」

→ **デメリットも先出し**しておくとトラブル回避になります。

## STEP3：応募文（提案文）のサンプル

### NGテンプレ

> はじめまして。GASができます。ぜひよろしくお願いいたします。

→ これは通りません。理由：**案件への言及ゼロ**だからです。

### OKテンプレ（コピペ可）

```
〇〇様

はじめまして、凛と申します。募集内容を拝見し、ぜひお力になれればと思い応募いたしました。

◆ 今回のご要望について
記載の「Gmailの特定ラベルを毎朝Slackに通知」については、GASの時間トリガー＋UrlFetchAppで実装可能です。類似の仕組みを自分の業務で運用しており、下記ポートフォリオがほぼ同じ構成です。

https://...（ポートフォリオURL）

◆ 想定する進め方
1. 要件ヒアリング（チャット15〜30分）
2. サンプルを1本動かして確認いただく
3. 本実装＋納品
4. 1週間の保守対応

◆ 納期・金額
2週間以内に納品可能です。ご予算〇〇円〜で調整させていただければと思います。

ご不明点あればお気軽にお知らせください。
どうぞよろしくお願いいたします。
```

### 応募文チェックリスト

| チェック項目 | OK? |

| 案件タイトルに具体的に触れているか | ☐ |
| 自分の過去成果物URLがあるか | ☐ |

| 納期・金額に触れているか | ☐ |

## STEP4：避けるべき地雷案件

| 案件の特徴 | なぜ避けるべきか |
|---|---|
| 発注者の評価が極端に低い／ゼロ | トラブル率が高い |
| 要件が「丸投げ」でゴールが不明確 | 時間がかかる割に評価が荒れる |
| 相場を大きく下回る（1件500円など） | 学習用以外は消耗する |
| チャット外で直接取引を誘ってくる | プラットフォーム違反で凍結リスク |

理由は単純で**母数が多いから**。「GAS」「Google Apps Script」「スプレッドシート自動化」「Gmail自動化」で検索すると、案件数が他社より体感で多い印象です。

## 初案件あるあるトラブルと対処

### ① 要件が途中で膨らむ

「ついでにこれも…」は必ず来ます。**最初に範囲を明文化**し、範囲外は追加見積もりで扱う旨を明示します。

### ② 返信が急に途絶える

発注者側の都合で止まることも頻繁にあります。**こちら側の予定を圧迫しないよう、複数案件を並行**しておくとリスクが分散します。

### ③ 支払いタイミングのズレ

プラットフォーム内の仮払い制度を必ず使う。**直接振込を持ちかけられたら原則お断り**で。

## 評価が貯まると何が変わるか

最初の3件を納品すると、プロフィールに数字の実績が付きます。そこから変わること：

- スカウトが届き始める
- 応募しなくても案件が流入してくる
- 相場より少し高い単価の案件に通りやすくなる

**最初の3件で世界が変わる**と思ってください。つまり、0→1さえ突破すれば、あとは加速していきます。

## 結論：今日やるべきは「1件の応募」

勉強も、ポートフォリオも、実は**2時間で最低限ができる状態**まで来ている人が大半です。足りないのは**応募ボタンを押す決心**だけ。

この記事を読み終わったら、クラウドワークスを開き、GASで検索し、**今日中に1件だけ応募**してみてください。2026年4月時点の私からのアドバイスとしては、それが一番効きます。
